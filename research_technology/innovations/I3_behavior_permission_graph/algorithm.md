# 机制设计

1. 对 Python、JavaScript/TypeScript 和配置文件构建安全解析器，解压前检查路径、符号链接、数量和体积。
2. 从 AST 提取进程执行、网络、敏感文件、动态加载、持久化等行为节点。
3. 构建跨文件调用图和敏感数据流，解析 manifest 与依赖锁文件生成声明权限和 SBOM。
4. 对“声明权限—实际行为—依赖能力”做一致性评分，输出文件、行、调用链和最小权限建议。
5. 使用固定版本的 CVE/恶意包快照离线复算，解析器强制超时和资源上限。

字符串扫描保留为 `src/baseline.py` 的 B1，不作为最终创新机制。B3 唯一入口为 `src/advanced_scanner.py`，依赖风险只使用版本化离线快照，扫描全程不 import、不运行目标代码。
