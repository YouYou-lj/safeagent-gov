# 机制设计

1. 规范化 `AuditEvent`，记录来源、身份、策略/模型/数据版本、审批角色和工具结果摘要。
2. 对规范化序列化结果计算 `event_hash`，并纳入 `prev_hash`、序号和 trace 标识形成链。
3. 校验器检查字段、内容哈希、顺序连续性、缺失事件和可选签名。
4. 回放包冻结输入、策略、模型参数、随机种子和工具模拟响应，重新执行裁决并比较输出摘要。
5. 审计写入失败时，高危动作失败关闭并记录可恢复告警。

当前 `src/audit.py`、`integrity.py` 和 `replay.py` 已实现 B3。旧 SQLite 记录只在整条 trace 均无哈希时一次性迁移；部分缺失不会自动修复。每次追加先校验当前链，回放包冻结策略文件内容与 SHA-256，并只复算安全裁决、读取记录的模拟响应摘要，不调用工具。
