# 技术方案

## 设计原则

系统以“不可信输入、默认拒绝、最小权限、人在回路、全程留痕”为基础。安全能力通过稳定字典契约解耦：输入检测输出风险与动作；工具网关输出决策与策略命中；所有模块只通过 trace 事件共享证据。

## 输入治理

PromptShield 读取 YAML 关键词和正则，按命中数量与来源加权评分。来源不仅是展示字段，也是分类依据：相同恶意指令出现在文档、网页、RAG 或记忆中时被归类为间接注入并隔离。代码保留轻量分类器/LLM 复核的插入点，但 MVP 不依赖外部模型。

## 执行治理

LangGraph 图依次执行 trace、inspect、plan、tools、finish。高风险输入条件分支直接进入 finish。所有工具经统一 `guarded_tool_call`，先记录申请，再做 YAML 策略裁决，只有允许类决策才进入模拟执行器。外部邮件必须审批，敏感路径/私网/未知工具/Shell/删除默认阻断。

## 供应链治理

SkillScan 对文本文件做静态令牌检测，核对 manifest 声明与实际行为，并安全处理 ZIP：拒绝绝对路径和 `..`，限制文件数、单文件大小和总解压大小。组件永不被 import 或运行。

## 审计与评测

SQLite 使用 `traces` 与 `audit_events` 两张真相表。事件只追加，JSON 保留模块原始决策。计划要求的
`task_trace`、`skill_execution_log`、`mcp_tool_log`、`sub_agent_log`、`model_call_log` 由五个只读 SQL View
从签名事件链投影，禁止写入形成第二份日志。报告导出不改变证据。评测脚本直接调用同一生产核心函数，避免
“展示逻辑”和“测评逻辑”漂移；分项 CSV 保留逐样例结果，汇总 JSON 供 Web 展示。

## 威胁边界

该原型证明安全链路和策略可行性，不是完整生产沙箱。当前已提供本地签名身份、租户/角色隔离、mode-0600 密钥文件、离线 CVE 快照和签名审计链，但尚未接入政企 OIDC/KMS、恶意软件动态分析、在线 CVE 情报、外部 WORM 日志或真实第三方模型对抗复核。生产部署必须补齐这些外部基础设施。
